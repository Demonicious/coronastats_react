Coronastats has shifted over to React.js


Many users reported that they were facing issues regarding the poor-performance of the website.

I was aware of this because, When I started development of this app.. The data-sets weren't really that big.
So I didn't bother to use "good coding practices" in my application, Which has been a catastrophic decision.
The data-sets became bigger and bigger, and the site couldn't keep up..

The site was built using Plain JavaScript with a little bit of jQuery. jQuery is a very old library and isn't recommended anymore, but I figured It'd be fine for the job... I was wrong!

Keeping this in mind, I decided to re-create the entire website using React.js which is a very popular
Front-end Building Library. I chose React because I was aware of it's many advantages in Single-page-apps & also that It would speed up developemnt time by a lot.

The following changes were made:

  • Website Completely Re-build in React.js.
  • Major overall performance improvements.
  • UI & UX Updates.
  • Changed Bar-charts to High-performance Line-charts.
  • Changed charting library from amCharts4 to better performing chart.js.
  • Old Map replaced with Highly-detailed & Highly-performant Leaflet Worldmap.
  • Replaced linear scales with Logarithmic scales.
  • Added more sorting options.
  • Improved performance of Country-Details Modals.
  • Made improvements to the Light-theme.
  • ... Lastly, Stay Safe & Thank you for the continued support!
